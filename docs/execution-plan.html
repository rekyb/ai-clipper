AI Clipper — Execution Plan

Date: 2026-05-17  |  Estimate: 21–27 days (~3-4 weeks)  |  Hardware: RTX 2000 Ada 8GB VRAM, 64GB RAM, Windows 11

Context & Goals

Problem: Creating viral short-form content from long-form videos is manual and expensive (online AI tools cost $30-100+/month).

Solution: A local, self-hosted AI pipeline using open-source models — unlimited clip generation at ~$0/month after setup.

Business goal: Generate extra monthly income by efficiently producing TikTok-ready clips.

Architecture

  • Frontend: Next.js 15 (TypeScript, MUI v7, pnpm)
  • Backend: Python FastAPI (transcription, LLM analysis, video processing)
  • AI Models: Whisper medium (always loaded) + Llama 3.1 8B Q5_K_M (always loaded)
  • Database: MongoDB (local)
  • Storage: File-based — media/originals/, media/exports/
  • Communication: REST API + WebSocket (real-time progress)

Constraints

  • 8GB VRAM budget: Whisper medium (~1.5GB) + Llama 8B Q5 (~2GB) + KV cache + overhead
  • Single-user local app — no auth, no scaling
  • Streaming pipeline: LLM analysis starts while Whisper still processes

3.1 LLM Service

  • Llama wrapper: load on startup, reuse for all analysis
  • GPU context: ~2GB VRAM + RAM offload via llama-cpp-python
  • Prompt engineering for viral scoring (JSON output with examples)
  • Criteria: hook strength, emotional arc, standalone comprehension, curiosity gap, rewatch potential

Key Files

  • backend/app/services/llm.py — Llama wrapper
  • backend/app/prompts/viral_scoring.txt — system prompt
  • backend/app/utils/scoring.py — weight calculation

3.2 Streaming Analysis Pipeline

  • Split transcript into 60s overlapping windows (30s overlap)
  • For each chunk: extract text → call LLM → parse JSON → score → save
  • Scoring weights: Hook 30% + Emotional 25% + Standalone 20% + Curiosity 15% + Rewatch 10%
  • Emit progress via WebSocket with new clips as they're found
  • Save clips to MongoDB with full score breakdown + reasoning

Key Files

  • backend/app/services/analysis.py — streaming orchestrator
  • backend/app/utils/transcript_chunking.py — windowing logic
  • backend/app/api/clips.py — GET clips endpoint

Risks & Mitigations

VRAM Contention High Impact Med Probability
Both models + inference may exceed 8GB. Mitigation: Q5 quantization, float16, RAM offload, VRAM monitoring, disable large-v3 if needed.
LLM Prompt Reliability Med Impact Med Probability
8B model may not always return valid JSON or accurate scores. Mitigation: Detailed prompt with examples, JSON validation, default score on parse error, iterative prompt tuning.
Poor Virality Score Accuracy Med Impact High Probability
AI-suggested clips may not actually be viral. Mitigation: Conservative threshold (≥60), user feedback loop, prompt iteration, treat as AI-assisted not magic.
Export Time Too Long Med Impact Med Probability
Export may take >10 min. Mitigation: FFmpeg preset "fast", H.264 (not HEVC), progress UI, background queue.
Video Format Compatibility Low Impact Med Probability
Unsupported codecs (VP9, AV1). Mitigation: Validate with ffprobe, re-encode if needed, document supported formats.
Caption Timing Misalignment Low Impact Low Probability
Captions drift from audio. Mitigation: Whisper large-v3 for re-transcription, preview before export.
