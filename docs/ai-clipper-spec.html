AI Clipper

Product Specification — v1.0 | 2026-05-17 | Local-first AI video clipping tool

1. Overview

Problem: Creating viral short-form TikTok content from long-form videos is time-consuming. Online AI tools are expensive for regular use.

Solution: A local web-based app that uses open-source AI models to automatically identify viral-worthy segments from long videos, score them, and export polished 30-60 second clips ready for TikTok.

Goal: Generate extra monthly income by efficiently producing viral short-form content from existing long-form videos.

User: Single user (developer), running locally on personal machine.

Key Architectural Decisions

  • Two-runtime split: Python handles AI/video natively; Next.js provides responsive UI
  • MongoDB: Stores job state, clip metadata, virality scores, export history
  • WebSocket: Real-time progress updates during processing
  • Both models always loaded: Whisper medium (~1.5GB VRAM) + Llama 8B Q5 (~2GB VRAM + RAM offload) coexist within 8GB VRAM budget
  • Streaming pipeline: LLM analysis starts on early transcript chunks while Whisper still processes later portions
  • File-based media storage: No blob store needed for single-user local app

4. AI Pipeline

Stage 1: Transcription

  • Model: Whisper medium (faster-whisper / CTranslate2)
  • Input: Video/audio file (any format)
  • Output: Word-level timestamped transcript with confidence scores
  • Performance: ~1-2 min for a 1-hour video
  • Features: Multi-language support, speaker detection

Stage 2: Viral Segment Analysis

  • Model: Llama 3.1 8B Q5_K_M (llama-cpp-python)
  • Input: Timestamped transcript in overlapping windows
  • Processing: Runs in parallel with Stage 1 (streaming pipeline)
  • Output: Ranked candidate clips with structured JSON

Virality Scoring (0-100)

FactorWeightDescription
Hook Strength30%Opening line grabs attention immediately
Emotional Arc25%Builds tension, surprise, humor, or insight
Standalone Comprehension20%Makes sense without full video context
Curiosity Gap15%Creates desire to watch until the end
Rewatch Potential10%Worth watching again or sharing

Stage 3: Video Processing

  • Tools: FFmpeg + OpenCV
  • Steps:
    1. Extract clip at identified timestamps (±2s padding)
    2. Smart crop to 9:16 (face/motion detection for crop position)
    3. Re-transcribe clip with Whisper large-v3 for precise word timing
    4. Generate animated word-by-word captions
    5. Burn captions into video
    6. Export as H.264 MP4 (TikTok-optimized: 1080x1920, 30fps)

5. Features (MVP)

Feature 1: Video Import MVP

  • Drag-and-drop or file picker for local video files
  • Supported formats: MP4, MKV, AVI, MOV, WebM
  • URL paste → auto-download via yt-dlp (YouTube, social media)
  • Display thumbnail, duration, file size after import
  • Store metadata in MongoDB

Feature 2: AI Transcription & Analysis MVP

  • Auto-transcribe on import with streaming progress (WebSocket)
  • LLM analyzes in parallel, identifies viral candidates
  • Results: ranked list of candidate clips by virality score
  • Each candidate shows: timestamp range, transcript preview, score breakdown, reasoning

Feature 3: Topic Search MVP

  • Text input to describe what you're looking for
  • LLM searches transcript, returns matching segments with relevance + virality scores
  • Combinable with auto-detected candidates (filter/re-rank)

Feature 4: Clip Preview & Selection MVP

  • Click candidate → video player seeks to timestamp
  • Drag handles to adjust start/end points
  • Preview clip in 9:16 with simulated caption overlay
  • Select/deselect clips for batch export

Feature 5: Export & Post-Processing MVP

  • Smart crop to 9:16 (face/motion-aware)
  • Re-transcribe with Whisper large-v3 for precise captions
  • Animated word-by-word captions (configurable font, size, color, position)
  • Export as TikTok-ready MP4 (1080x1920, H.264)
  • Save to media/exports/ with metadata

Feature 6: History & Library MVP

  • Dashboard of all processed videos and exported clips
  • Re-open past analyses without re-processing
  • Track export history per video

10. UI Layout (Wireframe Description)

  • Dashboard (/): Grid of processed videos with thumbnails, clip counts, last activity. Import button prominent.
  • Video View (/video/:id): Split layout — video player (top/left) + clip candidates list (right/bottom). Each clip card shows score, timestamp, preview text. Click to seek.
  • Clip Editor: Inline within video view. Timeline with drag handles, 9:16 preview panel, caption settings, export button.
  • Search: Text input above clip list. Results replace/filter the auto-detected candidates.
  • Export Queue: Bottom drawer showing export progress for selected clips.
